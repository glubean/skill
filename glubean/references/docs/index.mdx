Write verification tests in TypeScript. Run them in your editor, in CI, and against production with the same code.

Glubean is for teams that want API tests, workflow tests, traces, and run history without splitting work across five different tools.

```typescript
import { test } from "@glubean/sdk";

export const health = test("health", async (ctx) => {
  const baseUrl = ctx.vars.require("BASE_URL");
  const res = await ctx.http.get(`${baseUrl}/health`);

  ctx.expect(res.status).toBe(200);
});
```

## How Glubean works

```text
write verification locally  ->  keep in git  ->  run in CI  ->  upload when useful
```

- The SDK is where you author tests.
- The VS Code extension is the recommended local entry point for debugging, running tests, and inspecting results. You can start without it, but it is the fastest path for iterating on failures and understanding responses.
- The CLI handles project init (`npx glubean@latest init`) and CI automation (`npx glubean@latest run tests/`).
- Cloud stores uploaded runs, traces, metrics, and alerts. It does not execute your code.

## Why teams adopt it

- One codebase across local development, CI, and production verification
- First-class traces and result files instead of raw terminal noise
- `explore/` as your API collection (replaces Postman), `tests/` as your CI regression suite
- Git-safe, reviewable, conflict-free — no proprietary cloud needed
