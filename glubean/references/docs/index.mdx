# Glubean

Glubean is a workflow verification platform. Write tests in TypeScript, commit to git, run locally or in CI. Supports HTTP, GraphQL, gRPC, and browser automation through plugins.

The VSCode extension serves two roles: a **visual and review layer** for the Glubean workflow (run, inspect, debug), and a **code-first replacement for Postman** — your `explore/` folder is your API collection, versioned in git, shared with the team, no per-seat pricing.

The same TypeScript file works as both an API collection entry and a CI regression test. No export step, no format conversion, no tool fragmentation.

```ts
// Agent-generated workflow — runs as-is, no rewrite needed
import { test } from "@glubean/sdk";

test("auth-flow")
  .step("login", async ({ http, secrets, expect }) => {
    const data = await http
      .post("/auth/login", {
        json: { username: secrets.require("USER"),
               password: secrets.require("PASS") }
      }).json();

    expect(data.accessToken).toBeDefined();
    return { token: data.accessToken };
  })
  .step("get profile", async ({ http, log }, { token }) => {
    const me = await http
      .get("/auth/me", {
        headers: { Authorization: `Bearer ${token}` }
      }).json();

    log(`Hello ${me.firstName}`);
  });
```

## Start here

### With an AI agent (recommended)

Install the skill, then ask your agent to generate a verification workflow.

```bash
npx glubean config mcp        # MCP server — lets AI run and inspect tests
npx skills add glubean/skill   # Skill — teaches AI the SDK patterns
```

Then in your AI tool (Claude Code, Cursor, Codex):

```text
"create a smoke test for the users API"
```

The agent writes the test, runs it via MCP, reads structured failures, fixes, and reruns — all in one conversation. [See the full AI workflow →](/extension/generate-with-ai)

## How Glubean works

```text
prompt → draft → inspect → repair → keep → CI → cloud
                                      ↑
                          (or write manually here)
```

- **Prompt**: Describe what to verify. The agent uses your skill and OpenAPI context.
- **Draft**: Agent generates a TypeScript workflow with named steps, typed assertions, and shared state.
- **Inspect**: Run in the extension. Structured traces show exactly which step failed and why — not a wall of logs.
- **Repair**: Agent reads typed failures (`expected` vs `actual`), fixes, and reruns. No copy-pasting errors.
- **Keep**: The same file moves from `explore/` to `tests/` — zero migration, same runner.
- **CI**: `glubean run tests/` in any CI pipeline.
- **Cloud**: Upload runs for team dashboards, analytics, and alerts.

## Why teams adopt Glubean

- **Agent output is not throwaway** — the same file graduates from `explore/` to CI without rewriting
- **Structured failures, not log noise** — AI (and humans) can read typed `expected` vs `actual` instead of parsing stack traces
- **One codebase** across local development, CI, and production verification
- **`explore/` replaces Postman** — your API collection lives in git, not a proprietary cloud
- **Git-safe and reviewable** — TypeScript files, no binary formats, no merge conflicts
