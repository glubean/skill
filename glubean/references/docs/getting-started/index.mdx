# Start Here

If you are new to Glubean, follow this path in order:

1. [Extension Quick Start](/extension/quick-start)
2. [Run Your First Test](/getting-started/first-test)
3. [Connect CI](/getting-started/ci)
4. [Upload Your First Run](/getting-started/upload-to-cloud)

## What this path gives you

- One working local test
- A predictable Node project layout
- A CI command you can keep
- A clean handoff into Cloud when the team is ready

## Choose the shortest useful route

- If you want the fastest local loop, start with [Extension Quick Start](/extension/quick-start).
- If you want to see the full editor workflow first, read [Editor Experience](/extension/editor-experience).
- If you are a QA engineer, start with [Glubean for QA Teams](/extension/for-qa-teams).
- If you are wiring automation first, go to [CLI Overview](/cli) and use `npx glubean@latest`.
- If you already have tests running and only need uploads, go straight to [Upload Your First Run](/getting-started/upload-to-cloud).
